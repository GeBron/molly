# 2026-07-16 工作日志

## 修复登录 500 问题（molly Spring Boot 项目）

### 根因（真正导致"无响应"的原因）
登录接口 `POST /api/auth/login` 返回 500，前端表现为点击登录无跳转（实为 500 被兜底成"系统内部错误"）。
- **主因**：MyBatis Mapper 的 JOIN 查询存在**歧义列**。`RoleMapper.selectByUserId` 与 `PermissionMapper.selectByRoleIds` 写成 `SELECT r.<include refid="roleColumns"/>`，MyBatis 只把 `r.` 拼到第一个列上，导致 `deleted/created_at/updated_at/created_by/updated_by` 未加表别名；而 `sys_role`/`sys_user_role`、`sys_permission`/`sys_role_permission` 都含这些列 → `Column 'deleted' in field list is ambiguous` → SQL 异常。该查询在 `UserDetailsServiceImpl.loadUserByUsername`（认证阶段）和 `buildUserInfo` 中被调用，异常被 Spring Security 包装为 AuthenticationException 后在 `AuthService.login` 抛出，最终被 `GlobalExceptionHandler` 兜底成 500。
- **次因**：登录失败时把最长 845 字符的异常信息写入 `sys_login_log.message`（列仅 VARCHAR(255)）→ 之前还叠加了 `Data too long` 错误。

### 已修复文件
1. `src/main/resources/mapper/RoleMapper.xml` — `selectByUserId` 改为全限定列 `r.id, r.role_code, ...`。
2. `src/main/resources/mapper/PermissionMapper.xml` — `selectByRoleIds` 改为全限定列 `p.id, p.perm_code, ...`。
3. `src/main/java/com/demo/molly/service/AuthService.java` — `saveLoginLog` 增加 `message` 截断(≤255) 并整体 try-catch，写登录日志失败不再影响登录主流程；补充 slf4j logger。

### 验证结果
`mvn spring-boot:run` 后：admin/admin123 → 200 操作成功（含 menus/permissions）；错误密码 → 401。前端 login.html/auth.js/common.js 经核查**无需改动**（事件绑定、URL `/api/auth/login` 一致性、CORS、code===200 判断均正确）。

### 沙箱调试经验（可复用）
- 本沙箱注入 `SERVER__PORT` 环境变量，会让 Spring Boot 监听随机端口（如 51333）而非 8080；本地启动需用 `env -u SERVER__PORT mvn spring-boot:run -Dserver.port=8080`。
- Java stdout 被块缓冲，后台进程被杀后日志文件为空；排查 500 时临时在 `GlobalExceptionHandler.handleException` 里把完整堆栈 `e.printStackTrace` 写入 `/tmp/molly_exc.log`（强制 flush）最可靠。
