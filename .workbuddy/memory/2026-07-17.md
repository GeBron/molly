## 修复：修改用户状态报「缺少必要参数：status」 ✅

- 根因：前端 `toggleStatus` 用 `ajaxRequest({type:'PUT', data:{status:newStatus}})` 发送，common.js 将其序列化为 JSON 请求体（`Content-Type: application/json`）。但后端 `UserController.updateStatus` 用 `@RequestParam Integer status` 接收，而 `@RequestParam` 读不到 JSON 请求体字段，导致 Spring 抛 `MissingServletRequestParameterException` → 全局异常翻译为「缺少必要参数：status」。RoleController / PermissionController 的 `/status` 接口是同样写法，同病。
- 第一版方案（已否决）：新增共享 DTO `StatusUpdateDTO` + 三个控制器改用 `@RequestBody @Valid StatusUpdateDTO`。用户认为为单个 `status` 字段新建 DTO 不合理。
- 最终方案（轻量）：后端保持 `@RequestParam` 不变；仅改前端三处 `toggleStatus`，把 `status` 拼到 URL 查询参数上（`/users/{id}/status?status=N`），不再发 JSON 请求体。`@RequestParam` 即可从查询参数读到。改动文件：`src/main/resources/templates/users.html`、`roles.html`、`permissions.html` 的 `toggleStatus`。已删除 `StatusUpdateDTO.java`。
- 验证：`mvn -o clean compile` 通过（EXIT=0），源码无 `StatusUpdateDTO` 残留引用。
- 经验：类似「单字段状态切换」接口，前后端约定用查询参数比新建 DTO 更合适；改前端比改后端接口契约成本低。

## 修复：角色/用户管理页预加载冗余查询 ✅

- 问题：角色管理页进入时预请求 `/api/permissions` 完整权限树；用户管理页进入时预请求 `/api/roles` 全部角色。仅当点击「分配权限/角色」时才需要这些数据，预加载造成无意义请求并阻塞首屏表格渲染。
- 方案：改为懒加载 + 缓存。
  - `roles.html`：移除页面初始化时的 `loadPermissions()`，改为在 `openAssignPerms()` 中通过 `$.when` 并发请求角色详情和权限树；`loadPermissions()` 增加缓存，已加载则返回已解析的 Deferred。
  - `users.html`：移除页面初始化时的 `loadRoles()`，改为在 `openAssignRoles()` 中通过 `$.when` 并发请求用户详情和角色列表；`loadRoles()` 增加缓存。
- 改动文件：`src/main/resources/templates/roles.html`、`src/main/resources/templates/users.html`。
- 效果：首屏只渲染主表格，减少进入页面时的后端查询；权限边界更清晰，没有分配操作的用户不会触发权限/角色查询。

## 代码审查：PageController 首屏数据使用分析

- 文件：`src/main/java/com/demo/molly/controller/PageController.java`
- 结论：首屏 `initialData` 在当前模板中确实被使用了，但不同页面价值不同。
- 列表页（`users.html`、`roles.html`、`login-logs.html`、`operation-logs.html`）：
  - `initialData.list` 用于服务端渲染 `<tbody>` 首屏行；
  - `initialData.total` 用于 DataTables `deferLoading`，避免页面加载后再次发起初次 AJAX；
  - 属于「有实际用途」的优化，不是死代码；若追求简化可改为纯 AJAX 首屏加载，但会增加一次请求。
- 权限页（`permissions.html`）：
  - `initialData` 服务端渲染的权限树行会被 JS `loadPermissions()` 立即覆盖；
  - 属于冗余查询，可以直接清理：`PageController.permissions()` 中移除 `permissionService.tree()` 调用及 `initialData` 注入，模板中移除 `permRows` 服务端渲染片段，保留客户端 `loadPermissions()` 逻辑即可。
